Chapter 7 : Circles and fixed circles collision

7.1 Add and draw a fixed circle

At former chapters, we implemented circle-circle collisions. Now we would add more types of objects to the canvas. The first thing to add is fixed circle, which doesn't move at all and circles would rebound when collide with it.

This time we start with a single circle as the following:

<canvas id="myCanvas" width="400px" height="400px"></canvas>
<script>
  const circleArray = [
    {
      cx:-60,
      cy:-60,
      r:0,
      m:Math.PI*50*50, //mass = area
      vx:15,
      vy:30
    }
  ];

  const canvas = document.getElementById("myCanvas");
  const ctx = canvas.getContext("2d");
  ctx.font="12px sans-serif";
  ctx.textAlign="center";
  const frameRate=60;

  setInterval(function(){
    //circle position update
    for(const c of circleArray){
        c.cx += c.vx * 1 / frameRate;
        c.cy += c.vy * 1 / frameRate;
    }

    //circle check collision : find if d is smaller than (or equals to) r1+r2
    for(let i = 0; i < circleArray.length ; i++){
      const c1 = circleArray[i];
      for(let j = i; j < circleArray.length ; j++){
        const c2 = circleArray[j];
        const dx=c2.cx-c1.cx;
        const dy=c2.cy-c1.cy;
        if(dx * dx + dy * dy <= (c1.r + c2.r)*(c1.r + c2.r)){ //find if d is smaller than (or equals to) r1+r2
          const rel_c2vx = c2.vx - c1.vx;
          const rel_c2vy = c2.vy - c1.vy;
          if(rel_c2vx * dx + rel_c2vy * dy < 0){ //apply collision only when 2 circles are colliding instead of separating
            c1.vx += 2 * dx * c2.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
            c1.vy += 2 * dy * c2.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
            c2.vx -= 2 * dx * c1.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
            c2.vy -= 2 * dy * c1.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
          }
        }
      }
    }

    //clear all drawings
    ctx.clearRect(0,0,canvas.width,canvas.height);

    //draw x and y axis
    ctx.strokeStyle="lightgray";
    ctx.beginPath();
    ctx.moveTo(0,canvas.height/2);
    ctx.lineTo(canvas.width,canvas.height/2);
    ctx.moveTo(canvas.width/2,0);
    ctx.lineTo(canvas.width/2,canvas.height);
    ctx.stroke();
    ctx.strokeStyle="black";

    //draw border
    ctx.beginPath();
    ctx.rect(0,0,canvas.width,canvas.height);
    ctx.stroke();

    //draw circle
    for(let i=0;i < circleArray.length;i++){
        const c=circleArray[i];
        ctx.fillStyle=ctx.strokeStyle="black";
        ctx.beginPath();
        ctx.arc(canvas.width/2+c.cx,canvas.height/2-c.cy,c.r,0,2*Math.PI);
        ctx.stroke();
        ctx.beginPath();
        ctx.fillText(c.cx.toFixed(1)+","+c.cy.toFixed(1),canvas.width/2+c.cx,canvas.height/2-c.cy+16);
        ctx.fillStyle=ctx.strokeStyle="blue";
        ctx.moveTo(canvas.width/2+c.cx,canvas.height/2-c.cy);
        ctx.lineTo(canvas.width/2+c.cx+c.vx,canvas.height/2-c.cy-c.vy);
        ctx.stroke();
        ctx.beginPath();
        ctx.arc(canvas.width/2+c.cx+c.vx,canvas.height/2-c.cy-c.vy,2,0,2*Math.PI);
        ctx.fill();
        ctx.fillText(c.vx.toFixed(1)+","+c.vy.toFixed(1),canvas.width/2+c.cx+c.vx,canvas.height/2-c.cy-c.vy-4);
        ctx.fillStyle=ctx.strokeStyle="gray";
        ctx.fillText("C"+i,canvas.width/2+c.cx,canvas.height/2-c.cy);
        ctx.fillStyle=ctx.strokeStyle="black";
        ctx.beginPath();
        ctx.arc(canvas.width/2+c.cx,canvas.height/2-c.cy,2,0,2*Math.PI);
        ctx.fill();
    }
  },1000/frameRate);
</script>

7.2 Circle and fixed circle collision response

So how would we simulate movablle circle and fixed circle collision? The idea is : when a movable circle collides with a fixed circle, it behaves as if colliding another movable circles with infinity mass! What does it mean?

First, let's see the following 3 canvas, which the mass (c1.m) of circle 1 (C1) increases from left to right:

Mass of circle 1 (C1) = π202 Mass of circle 1 (C1) = π202 * 10 Mass of circle 1 (C1) = π202 * 100

We can see, when the mass of circle 1 (C1) increases, circle 2 (C2) becomes moving slower after collision. At the last canvas, circle 1 (C1) almost becomes not moving, which looks as if circle 2 (C2) colliding with a fixed obstacle!

So the concept : assume m of circle 1 (C1) is infinity! Let's start coding for fixed circles! Recall the collision response formulae of circle 1 (C1) before:

Circle 1 (C1):
c2.vx = c2.vx - 2 * dx * c1.m * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (c1.m + c2.m)
c2.vy = c2.vy - 2 * dy * c1.m * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (c1.m + c2.m)

We may simulate fixed circles as if a movable circle with infinity mass. Substitute infinity into the formula like the following:

Circle 1 (C1):
c2.vx = c2.vx - 2 * dx * ∞ * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (∞ + c2.m)
c2.vy = c2.vy - 2 * dy * ∞ * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (∞ + c2.m)

But wait! Isn't the new c2.vx and c2.vy becomes ∞/∞?

Not really! But first, we need to slightly transform the formulae: divide both the upper and lower part of the fraction by c1.m:

Circle 1 (C1):
c2.vx = c2.vx - 2 * dx * c1.m * (rel_vx * dx + rel_vy * dy) / c1.m
(dx * dx + dy * dy) * (c1.m + c2.m) / c1.m
c2.vy = c2.vy - 2 * dy * c1.m * (rel_vx * dx + rel_vy * dy) / c1.m
(dx * dx + dy * dy) * (c1.m + c2.m) / c1.m

Slightly change the position of the term '/ c1.m':

Circle 1 (C1):
c2.vx = c2.vx - 2 * dx * c1.m / c1.m * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (c1.m / c1.m + c2.m / c1.m)
c1.vy = c1.vy - 2 * dy * c1.m / c1.m * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (c1.m / c1.m + c2.m / c1.m)

Simplify the formulae above:

Circle 1 (C1):
c2.vx = c2.vx - 2 * dx * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (c1.m / c1.m + c2.m / c1.m)
c2.vy = c2.vy - 2 * dy * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy) * (c1.m / c1.m + c2.m / c1.m)

The core of this section is here : When c2.m tends to infinity, c2.m / c1.m tends to 0 (because c2.m is finite), as if 1 / ∞ tends to 0! So the formulae can furthur simpified as:

Circle 1 (C1):
c2.vx = c2.vx - 2 * dx * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy)
c2.vy = c2.vy - 2 * dy * (rel_vx * dx + rel_vy * dy)
(dx * dx + dy * dy)

Recall the collision detection and response handling code of 2 movable circles :

//circle check collision : find if d is smaller than (or equals to) r1+r2
for(let i = 0; i < circleArray.length ; i++){
  const c1 = circleArray[i];
  for(let j = i; j < circleArray.length ; j++){
    const c2 = circleArray[j];

    const dy=c2.cy-c1.cy;

      const rel_c2vx = c2.vx - c1.vx;
      const rel_c2vy = c2.vy - c1.vy;
      if(rel_c2vx * dx + rel_c2vy * dy < 0){ //apply collision only when 2 circles are colliding instead of separating
        c1.vx += 2 * dx * c2.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
        c1.vy += 2 * dy * c2.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
        c2.vx -= 2 * dx * c1.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
        c2.vy -= 2 * dy * c1.m * (rel_c2vx * dx + rel_c2vy * dy) / (dx * dx + dy * dy) / (c1.m + c2.m);
      }
    }
  }
}

To change it to circle and fixed circle collision code, We would modify it by:

  • Replacing the for loop that circle 1 (C1) vs circle 2 (C2) with for loop that circle vs fixed circle
  • Replacing rel_vx with c2.vx also rel_vy with c2.vy, because fixed circle has no velocities
  • Edit the collision response formula

The circle and fixed circle collision code becomes:

//fixed circle and circle check collision
for(const fc of fixedCircleArray){
  for(const c2 of circleArray){
    const dx=c2.cx-fc.cx;
    const dy=c2.cy-fc.cy;
    if(dx * dx + dy * dy <= (fc.r + c2.r)*(fc.r + c2.r)){ //find if d is smaller than (or equals to) r1+r2
      if(c2.vx * dx + c2.vy * dy < 0){ //apply collision only when 2 circles are colliding instead of separating
        c2.vx -= 2 * dx * (c2.vx * dx + c2.vy * dy) / (dx * dx + dy * dy);
        c2.vy -= 2 * dy * (c2.vx * dx + c2.vy * dy) / (dx * dx + dy * dy);
      }
    }
  }
}

But wait! The code has a problem: c2.vx would be modified before c2.vy updates, which c2.vy needs old c2.vx to update!

In order to solve the problem, we would create a new variable : vxFactor (indicate it is related to vx), to store the value of c2.vx * dx + c2.vy * dy first:

//fixed circle and circle check collision
for(const fc of fixedCircleArray){
  for(const c2 of circleArray){
    const dx=c2.cx-fc.cx;
    const dy=c2.cy-fc.cy;
    if(dx * dx + dy * dy <= (fc.r + c2.r)*(fc.r + c2.r)){ //find if d is smaller than (or equals to) r1+r2
      const vxFactor = c2.vx * dx + c2.vy * dy;
      if(vxFactor < 0){ //apply collision only when 2 circles are colliding instead of separating
        c2.vx -= 2 * dx * vxFactor / (dx * dx + dy * dy);
        c2.vy -= 2 * dy * vxFactor / (dx * dx + dy * dy);
      }
    }
  }
}

7.3 Circle and fixed circle internal (inner shell) collision response

At last section, we implemented collision between circle and outer shell of fixed collision. To reuse the codes and add more features, we may add the code of collision between circle and inner shell of fixed collision.

    //draw fixed circle

        ctx.beginPath();
        ctx.arc(canvas.width/2+fc.cx,canvas.height/2-fc.cy,2,0,2*Math.PI);

    //draw circle
    for(let i=0;i < circleArray.length;i++){
        const c=circleArray[i];

        ctx.stroke();

        ctx.fillText(c.cx.toFixed(1)+","+c.cy.toFixed(1),canvas.width/2+c.cx,canvas.height/2-c.cy+16);
        ctx.fillStyle=ctx.strokeStyle="blue";
        ctx.moveTo(canvas.width/2+c.cx,canvas.height/2-c.cy);
        ctx.lineTo(canvas.width/2+c.cx+c.vx,canvas.height/2-c.cy-c.vy);
        ctx.stroke();
        ctx.beginPath();
        ctx.arc(canvas.width/2+c.cx+c.vx,canvas.height/2-c.cy-c.vy,2,0,2*Math.PI);
        ctx.fill();
        ctx.fillText(c.vx.toFixed(1)+","+c.vy.toFixed(1),canvas.width/2+c.cx+c.vx,canvas.height/2-c.cy-c.vy-4);
        ctx.fillStyle=ctx.strokeStyle="gray";
        ctx.fillText("C"+i,canvas.width/2+c.cx,canvas.height/2-c.cy);
        ctx.fillStyle=ctx.strokeStyle="black";
        ctx.beginPath();
        ctx.arc(canvas.width/2+c.cx,canvas.height/2-c.cy,2,0,2*Math.PI);
        ctx.fill();
    }
